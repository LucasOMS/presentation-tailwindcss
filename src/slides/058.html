Starting styles

Utilitaire

<div class="starting:opacity-0">...</div>
div {
    @starting-style {
        opacity: 0;
    }
}

Transitions

  • transition-colors, transition-opacity, transition-transform, transition-all, transition-none
  • duration-300, valeur en ms, peut varier. duration-initial pour réinitialiser
  • delay-300, valeur en ms, peut varier.
  • transition-discrete 
En 2023, CSS sortait la fonction de "starting style", cela permet de définir un style qui n'est appliqué qu'au moment du premier rendu dans le DOM de l'élément. Combiné à une transition, cela permet de facilement créer des animations d'entrée.

→

Concernant le support, le caniuse est autour des 88%. Les principaux navigateurs le supportent depuis quelques années, évidemment, si vous ciblez Internet Explorer, vous êtes foutus :)

→

Puisque cela est une "at rule" de CSS, Tailwind le traite comme un variant. Il suffit de rajouter starting: devant l'utilitaire et il ne sera appliquée que quand l'élément est rendu une première fois. Si vous voulez l'utiliser avec des transitions, il faudra utiliser les utilitaires de tailwinds pour les transitions.

→

Voilà les utilitaires principaux qui sont liés aux transitions et animations. D'abord il faut pouvoir spécifier sur quoi on active les transitions, par défaut, on a quelques valeurs, sinon il faudra utiliser le "all" ou bien une valeur arbitraire pour être plus spécifique.

Ensuite vous pouvez spécifier la durée le la transition. Biensur en faisant comme ca, ca spécifie une seule durée pour toutes les transitions.

Si vous avez besoin d'ajouter un délai, vous pouvez utiliser l'utilitaire delay-<durée>

Enfin, si vous avez besoin de transitions ou d'animation sur des valeurs qui ne sont pas linéaires, vous pouvez spécifier transition-behavior: allow-discrete avec l'utilitaire transition-discrete
