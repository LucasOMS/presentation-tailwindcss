Petit mot rapide pour parler d'une migration depuis Bootstrap vers Tailwind.

→

Déjà, il va falloir anticiper les configurations que vous allez devoir mettre en place.
Cela inclue la palette de couleur, l'espacement, les tailles de texte et les arrondis, qui peuvent différer de Bootstrap.
Le reset CSS de Bootsrap est celui de Tailwind diffère également légèrement, il faudra donc le prendre en compte, notemment sur le texte

→

Vous pourrez ensuite trouver en ligne des outils qui permettent de convertir des templates Bootstrap en Tailwind mais ils ne permettent pas de faire tout le projet automatiquement.

→

Vous pouvez utiliser une approche par Regex pour remplacer au fur et à mesure. Cela fonctionnera bien pour tout ce qui est espacement, arrondi, taille, et layout en général. C'est vrai pour la plupart des layout à part la grille CSS. Celle-ci étant très spécifique, sa conversion créera souvent de nommbreuses classes utilitaires, notamment pour récréer les marges et les espacements.

Pour utiliser des regex un peu plus avancé qui cible spécifiquement les classes dans votre HTML, je vous parlerez d'un des mes projets, un plugin eslint qui permet de cibler et remplacer des classes efficacements avec une syntaxe simple.

De facon globale, une conversion créera souvent de nombreux utilitaires relativements peu lisibles ou cohérents, notemmant pour reproduire les différentes marges créées par Bootstrap.
