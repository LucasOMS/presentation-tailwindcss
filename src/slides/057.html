Container queries

Créer un container

<div class="@container">...</div>

Créer un container nommé

<div class="@container/card">...</div>

Cibler en fonction de la taille

<div class="p-2 @sm:text-left @sm/card:p-1">...</div>

Cibler une taille maximale

<div class="@max-lg:text-left">...</div>

Cibler une plage de taille

<div class="@sm:@max-lg:text-left">...</div>

Cibler une valeur arbitraire

<div class="@min-[300px]:text-left @max-[600px]:p-1">...</div>
Tailwind permet de créer des utilitaires pour les containers queries. En v3 c'était un plugin, dans la version 4, c'est fourni de base.

Leur utilisation va ressembler à celle des groupes pour les hover, focus, etc.

→

On va pouvoir créer un container avec la classe @container.

→

@container/nom nous permet de créer un container nommé.

→
Pour cibler une taille de container au lieu d'une taille d'écran, on va ajouter un @ devant la taille. Si on cible un container nommé, on ajoute son nom après la taille avec un slash.
Par défaut les tailles sont les mêmes que celles pour les écrans, mais ce ne sont pas les mémes variables qui sont impliquées, vous pouvez donc les personnaliser indépendamment.

→
Comme pour la taille d'écran, il est possible de cibler une taille maximale en préfixant la taille par @max-.

→
Il est possible de cibler une plage de taille en combinant les deux, avec @<taille-min> et @max-<taille-max>.

→
Si on veut cibler une taille en étant au pixel prêt, il est possible d'utiliser une valeur arbitraire en utilisant les préfixes @min et @max suivis de la valeur exacte entre crochets.
Il faut se rappeler que ces valeurs seront utilisés dans des containers queries, il est donc impossible d'utiliser des variables CSS ici.

Les containers queries sont en cours d'implémentations dans les navigateurs, il y a des brouillons actuellement pour permettre de cibler le style du container. Pour le moment il n'y a aucun support dans tailwind, je n'ai vu aucune communication sur le sujet, je ne sais pas si cela arrivera comme un plugin ou si ce sera pour une future version. De toute facon, c'est probablement trop jeune pour pouvoir être utilisé en production.
