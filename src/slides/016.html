Les arguments contre les utilitaires

Taille du HTML

  • Factorisation des utilitaires
  • Composants (framework) 
La taille du HTML quand on utilise beaucoup d'utilitaire peut en effet être un argument en défaveur des classes utilitaires.

→
Comme je l'ai évoqué pour la taille du CSS, il est possible de faire des utilitaires qui sont des raccourcis pour un ensemble, cela permet de réduire le nombre de classe appliquée.

→
Enfin dans un contexte avec des framework front-end, on ne charge le HTML d'un composant qu'une seule fois sur le réseau, ce n'est donc pas si couteux in fine.
