Personnaliser Tailwind

Fonctions et directives

Directive @apply 

h1 {
    @apply text-3xl font-bold text-cyan-950 md:text-5xl lg:text-7xl dark:text-cyan-200;
}

Référencer le theme

h1 {
    margin-bottom: theme(spacing.1);
    margin-top: theme(spacing[0.5]): /* [] car variable avec un point */
}
h1 {
    margin-bottom: var(--spacing);
    margin-top: calc(var(--spacing) * 0.5);
}

Fonction --spacing()

N'existe pas !
h1 {
    margin-bottom: --spacing(1);
    margin-top: calc(var(--spacing) * 0.5);
}
<div class="py-[calc(--spacing(4)-1px)]"></div>
En v3 comme en v4, on peut utiliser la directive @apply dans le CSS pour appliquer le style d'un utilitaire.
Ici par exemple en une ligne j'applique de nombreuse règle et je vais même jusqu'à appliquer des styles en fonction du theme et des breakpoints.
Cela permet de facilement conserver la cohérence du style personnalisé et du style de Tailwind.
Il faut savoir qu'utiliser @apply juste pour rendre le CSS "plus court" ou "plus lisible" va quand même à l'encontre de la philosophie de Tailwind, puisque cela revient quand même à écrire du CSS.

→

Lorsqu'on fait du CSS à la main, on peut vouloir référencer le theme de Tailwind pour appliquer des styles.

En v3 on peut utiliser la fonction theme() qui sera interprétée par Tailwind lors de la compilation.

V

En v4, on peut toujours utiliser ca mais c'est déprécié, il est recommandé d'utiliser directement la variable CSS dont on veut la valeur.

→

Ici vous voyez que j'ai utilisé calc avec la variable, mais dans le cas du spacing, on peut utiliser la fonction --spacing de tailwind qui prend en argument le multiplicateur.

Cela permet d'être plus efficace, et cette fonction peut également être utilisée dans des utilitaires
