Appliqué

<card class="p-6"></card>

Non appliqué

<card class="p-2"></card>

Correction : style appliqué

<card class="p-2!"></card>
Dans certains cas, utiliser des classes utilitaires peut rendre difficile le fait de changer un style appliqué par défaut sur un composant.

→

Si on reprend notre exemple de card de tout à l'heure, j'applique par défaut un padding de 4 espacements. Mettons que je veuille une card avec un padding de 6 espacements à la place, je peux appliquer un utilitaire p-6 →. Cela fera que j'aurais une classe p-4 et p-6, p-6 étant défini plus loin que p-4 dans le code, j'ai bien un padding de 6 espacements.

→

Maintenant si je veux RÉDUIRE le padding, et que j'applique la classe p-2, vu qu'elle est défini avant dans le code, je ne peux pas le faire.

→

Pour bootstrap, cette problématique est vrai, et on serait obligé de passer par du style inline ou un fichier CSS avec un sélecteur plus précis, ou bien utiliser !important. Dans tailwind, il est possible de spécifier ce !important directement dans l'utilitaire, afin de pouvoir utiliser cette mécanique de surcharge.
