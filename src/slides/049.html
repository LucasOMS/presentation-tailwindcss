Ajouter tailwind à un projet

Avec la CLI

  • Installer les dépendances
  • Ajouter le fichier de configuration : JS ou CSS
  • Lancer tailwind avec la ligne de commande
  • Ajouter le fichier CSS généré dans votre projet

Avec PostCSS

  • Installer les dépendances
  • Ajouter tailwind en tant que plugin postcss
  • Importer tailwind dans votre CSS
  • Ajouter le fichier CSS généré dans votre projet

Play CDN

  • Ajouter le script dans le HTML
  • Ajouter une logique pour lui donner vos configurations
Pour le développement seulement

Les frameworks

RTFM !
Je passe rapidement sur les différents moyens d'ajouter Tailwind, biensur le plus efficace est d'aller directement voir leur documentation et suivre ces étapes.

On peut utiliser la CLI de tailwind, dans ce cas, ca veut dire que tailwind ne se lancera que quand on utilisera une ligne de commande spécifique. Il faudra donc passer par un autre outil pour que tailwind se lance lorsque vos fichiers sont modifiés. C'est pas exemple ce qui est utilisé pour la réalisation de cette présentation, j'ai une tache Gulp qui lance tailwind quand je modifie un de mes fichiers HTML, la configuration de tailwind, où les fichiers qui peuvent contenir des classes utilitaires.

→
On peut utiliser tailwind en tant que plugin postcss. Dans ce cas si postcss est configuré dans notre projet, tailwind sera lancé en même temps. C'est par exemple ce qui est préconisé pour l'utilisation dans des frameworks comme Angular ou Next.js.

→
Tailwind propose également un script qui permet de charger tailwind dynamiquement et qui analyse constamment notre DOM pour générer les éventuels nouveaux utilitaires. Il ne devrait être utilisé qu'en développement.

Je l'évoquais tout à l'heure, travailler en live avec tailwind peut être interessant. Si vous le souhaitez vous pouvez créer une configuration de dev qui charge le CDN, met à disposition votre configuration tailwind en tant que fichier statique afin de le charger au lancement de votre application. Cela vous permettra d'utiliser tailwind en live tout en ayant une configuration commune à celui de votre projet.

→
Pour tous les frameworks, il faut directement consulter la doc, il y a plus d'une quinzaine de frameworks supportés, et la configuration est souvent très simple.
