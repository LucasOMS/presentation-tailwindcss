Inhérent aux frameworks
Avoir autant de classes utilitaires semble comparable à apprendre un nouveau langage. Beaucoup d'utilitaire doivent être appris pour pouvoir les utiliser. Dans bootstrap par exemple, beaucoup de propriétés sont abrégés, il faut savoir que pour le display, c'est d-. Certains raccourcis n'existant pas en CSS, il faut les apprendre, comme px et py pour le padding par exemple

→

C'est un argument valable et assez difficilement démontable. C'est le probleme de tout framework, ca demande un apprentissage. Mais sur un projet utilisant du CSS, il faut apprendre les variables SCSS, les variables CSS, les mixins. Il y a toujours quelque chose à apprendre. Concernant tailwind, on a une documentation facilement consultable et assez efficace, si on recherche une propriété à définir, on tombe toujours sur l'utilitaire qui lui est lié. Si ce n'est pas le cas, c'est qu'il n'existe pas encore. Il est donc possible de le créer.



On verra que dans la syntaxe tailwind est assez facile à appréhender pour ceux qui connaissent CSS, pour les utilitaires de raccourci, c'est un peu d'apprentissage, un peu accéléré si on connait Bootstrap.
