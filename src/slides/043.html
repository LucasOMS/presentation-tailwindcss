const baseSpacing = 8; // px

Espacement

/* tailwind.css */
@import "tailwindcss"

@theme {
    --spacing: 8px;
}
Puisque tailwind dépends de nombreuse variables, il est possible de les personnaliser ou d'en ajouter.

→

Pour les couleurs par exemple, on peut utiliser la valeur colors du theme. Puisqu'on est en JS, il est possible en v3 d'utiliser directement les valeurs par défaut. Par exemple ici je défini une couleur primary en utilisant la couleur violette de Tailwind.

V

En v4, on va définir un bloc @theme dans lequel on va définir des variables CSS. Ces variables doivent suivrent un certains format, et c'est ce format qui va permettre à tailwind de les utiliser dans ses utilitaires.

Pour réutiliser du code existant, on peut soit utiliser les variables qui auraient été générées, soit retrouver la valeur sur la doc de tailwind.

Entre tailwind v3 et v4, la palette de couleur a légèrement changé, je vous conseille donc d'éviter de dépendre de leurs couleurs par défaut.

→

On peut également redéfinir l'espacement, via la variable spacing. Ici je défini seulement l'espacement de base à savoir 8px pour moi.

V

En v3 j'utilise la valeur spacing, et je défini les différents espacements utilisables. En v4 je n'avais pas eu à le faire car une fois le spacing défini, on peut utiliser les utilitaires avec des valeurs flottantes et tailwind les utilisent telles quelles dans le CSS via la fonction calc() de CSS.
