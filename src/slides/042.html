Configuration

/* tailwind.css */
@import "tailwindcss" prefix(tw);

Utilisation

<div class="hover:tw-bg-gray-500"></div>
Il est possible de définir un préfixe pour les utilitaires tailwinds, cela oblige tous les utilitaires à commencer par ce préfixe.

Il y a plusieurs différences entre la version 3 et la version 4.

→

En v4, c'est au moment de l'import que ca se joue. Cette fois le préfixe est considéré comme une variant qui doit être utilisée avant toutes les autres. Cela veut dire que vous aurez toujours préfixe:

Lorsque vous utilisez un préfixe, il n'est pas nécessaire de préfixer les variables dans la configuration de tailwind. En revanche les variables CSS générées utiliseront ce préfixe afin de ne pas rentrer en conflit avec d'autres variables CSS de votre projet.

V

En v3, il suffit de définir la propriété prefix dans la configuration. Le préfixe doit être utilisé comme tel.

Il faut considérer que le préfixe s'applique avant le nom de la propriété de l'utilitaire. Cela veut dire que le préfixe se retrouvera après les variants comme hover: par exemple.

Toute classe que vous définirez n'utilisera pas le préfixe par défaut, il faudra le respécifier.
