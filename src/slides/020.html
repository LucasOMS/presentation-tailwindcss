Les avantages des utilitaires

Aperçu rapide de la page

Définir le layout

Sans utilitaire

<nav>
    <ul>
        <li>Accueil</li>
        <li>Options</li>
    </ul>
</nav>
nav ul {
    display: flex;
    flex-direction: row;
    gap: 1rem;
}

Avec utilitaires

<nav>
    <ul class="flex flex-row gap-1">
        <li>Accueil</li>
        <li>Options</li>
    </ul>
</nav>

S'adapter aux écrans

Sans utilitaire

<nav>
    <ul>
        <li>Accueil</li>
        <li>Options</li>
    </ul>
</nav>
nav ul {
    display: flex;
    flex-direction: column;
    gap: 1rem;
}

@media (width >= 576px) {
    nav ul {
        flex-direction: row;
    }
}
@media (width >= 768px) {
    nav ul {
        gap: 2rem;
    }
}

Avec utilitaires

<nav>
    <ul class="flex gap-1 sm:flex-row md:gap-2">
        <li>Accueil</li>
        <li>Options</li>
    </ul>
</nav>
L'autre gros point positif des utilitaires, c'est la facilité qu'on a avoir un apercu mental de la page à partir de l'HTML.

→

En effet, pour le layout simple, c'est assez agréable de l'avoir directement dans le HTML, ici dans mon exemple, c'est impossible de prévoir que la liste est horizontale quand c'est fait en CSS pur.

→

Pareil pour les breakpoints, avec tailwdind, qui préfixe les classes avec le nom du breakpoint, ca permet de très rapidement voir quels sont les breakpoints appliqués, et au fur à mesure de la lecture des classes, savoir ce qui change entre la version mobile et la version desktop.

Alors, on pourrait se dire que c'est vrai que si on a pas trop de classe sur un meme élément, et je suis d'accord avec vous. Mais je voudrais vous montrer l'exemple qui était présent sur la doc de tailwind 3 (en page d'accueil hein !)
