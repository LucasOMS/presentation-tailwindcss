Javascript dynamisme 
    Un autre grand tournant dans le monde a été la création de Bootstrap et sa mise à disposition auprès de développeurs pas toujours à l'aise avec CSS. Alors, Bootstrap, c'était en 2011, donc il va falloir être indulgent envers le design. Nous allons voir quelques exemples de composants fournis par Bootstrap.
    →
    Des composants ultra simples d'utilisation via une seule classe sur l'élément
    →
    Des boutons assez simples à utiliser, ils nécessitent tout de même d'utiliser plusieurs classes en fonction de ce que l'on veut : le rendu de couleur, la taille
    →
    D'autres composants comme les formulaires sont plus compliqués à utiliser. En voyant ce code HTML, si vous n'avez pas fait de Bootstrap, vous ne comprenez probablement plus trop, c'est normal. Si certains composants sont simples à utiliser comme les tableaux, puisque la sémantique force une certaine structure dans le HTML (thead, tbody, etc.), d'autres composants nécessitent de connaître d'une part les classes CSS à appliquer mais également la structure HTML qui doit leur correspondre. Et ça, c'est d'autant plus vrai dans l'utilisation de la grille de Bootstrap, l'utilitaire le plus utilisé du framework.

    →

    Donc, si l'on reprend notre séparation des préoccupations, on commence à fortement lier HTML et CSS → puisque la structure devient ici essentielle au bon fonctionnement de notre design.

    

    C'est dans ce contexte que Bootstrap renonce entièrement à la séparation des préoccupations et fournit des classes CSS qui n'ont plus de sens "métier" mais uniquement un sens graphique.
